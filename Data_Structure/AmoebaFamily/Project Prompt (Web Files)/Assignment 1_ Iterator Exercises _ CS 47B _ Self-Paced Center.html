Assignment 1: Iterator Exercises

Goals

These exercises familiarize you with the design and use of an iterator, a class used to process all the elements in a collection one by one.

Readings

  • Data Structures and Problem Solving using Java (4th Edition) Sections 6.2 - 6.3
  • Data Structures and Problem Solving using Java (4th Edition) Sections 17.1 - 17.2, 17.5
  • Data Structures and Problem Solving using Java (4th Edition) Section 18.4
  • Data Structures and Problem Solving using Java (4th Edition) Chapter 15

Background

An amoeba family tree is simpler than a normal family tree because amoebas do not get married. An amoeba has one parent, perhaps a million siblings (brothers and sisters), and billions and billions of children. An amoeba also has a name.

You supply code to allow the amoebas in the family to be processed.

Files

  • AmoebaFamily.java

Exercise 1

Provide the body for the AmoebaFamily.addChild method. Given a name of the parent amoeba and the name of a new child, addChild should return having added a new amoeba as the youngest child of the named parent amoeba. Do not change any of the framework code; you may, however, supply additional methods. Also, do not use the ArrayList.get method; instead, use the ArrayList.iterator() method to iterate over each amoeba's children.

Exercise 2

Provide the body for the AmoebaFamily.print() method. It should print the names of all amoebas in the family in preorder, with children's names printed oldest first. Output for the family set up in the AmoebaFamily.main method should be

Amos McCoy
mom/dad
me
Mike
Bart
Lisa
Homer
Marge
Bill
Hilary
Fred
Wilma

The print method or one of the methods it calls should be recursive. It should print the names of an amoeba family of N amoebas in O(N) operations (counting each call to Amoeba.toString() as a single operation). Do not change any of the framework code; you may, however, supply additional methods.

Test your code for Exercises 1 and 2 using the AmoebaFamily.main method.

Exercise 3

Complete the AmoebaIterator class. It should successively return amoebas from the family in preorder, with children amoebas returned oldest first. Thus, for the family set up in the AmoebaFamily.main method, the amoeba named "Amos McCoy" should be returned by the first call to next(); the second call to next() should return the amoeba named "mom/dad"; and so on. Do not change any of the framework code.

Iteration over all of the amoebas in a family of N amoebas should require no more than O(N) operations. Also, the iterator constructor should run in constant time (thus, a solution that puts all the amoebas into a linear structure, then returns them one by one is not allowed). Your implementation will require the use of a Stack.

Uncomment the code at the end of the AmoebaFamily.main method to test your solution. It should produce the same output as the call to family.print().

Exercise 4

Rewrite the AmoebaIterator class so that amoebas are returned breadth first. That is, the root of the family should be returned first, then all of its children (oldest first), then all of their children, and so on. For the family constructed in the AmoebaFamily.main method, the amoebas should be returned in the following sequence:

Amos McCoy
mom/dad
me
Fred
Wilma
Mike
Homer
Marge
Bart
Lisa
Bill
Hilary

As in Exercise 3, enumeration of all the amoebas in a family of N amoebas should require no more than O(N) operations, and the constructor must run in constant time. Your implementation will require the use of a Queue where the Exercise 3 solution used a Stack.

Test your code with the AmoebaFamily.main method from Exercise 3.

Checklist

  • Correctly working additions to the framework, implemented without changing any of the framework code:
    • an addChild method that uses ArrayList.iterator() to examine amoeba children;
    • an efficient print() method that prints amoeba names in the specified order;
    • one version of the AmoebaIterator class in which the next() method efficiently returns amoebas in preorder, with children returned oldest first;
    • another version of the AmoebaIterator class in which the next() method efficiently returns amoebas breadth first;
    • constructors for these iterators that run in constant time.
  • Program listings, with methods indented to show their structure, accompanied by comments that describe their purpose and arguments.
  • Reasonable names for variables, methods, and parameters.
